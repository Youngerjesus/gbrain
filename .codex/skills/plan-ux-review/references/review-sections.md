# Plan UX Review Sections

Run all eight passes after Step 0 investigation. If a pass has no findings, say so and explain why. Every pass must connect to the target user, job-to-be-done, first-value target, scenario set, or journey trace.

## Pass 1: First Value And Onboarding

Rate 0-10: Can the target user reach the first meaningful value quickly and confidently?

Evaluate:

- entry point matches user intent
- first screen or first step explains what is happening
- the product asks only for necessary information
- setup cost is proportional to expected value
- progress or relief appears early
- next action is unmistakable

Plan obligation for 10/10: exact first-value moment, steps to reach it, copy or cues that confirm progress, and a target time or step count.

## Pass 2: Core Task Flow

Rate 0-10: Can the user complete the main job without unnecessary detours?

Evaluate:

- start, middle, and finish are explicit
- primary action is clear at each stage
- task can be completed without reading unrelated material
- system feedback keeps the user oriented
- completion state tells the user what changed
- follow-up action is natural

Plan obligation for 10/10: scenario-level flow including user actions, system responses, and completion criteria.

## Pass 3: Information Architecture

Rate 0-10: Is information organized around user decisions and attention?

Evaluate:

- the most important information appears before supporting detail
- labels match user language
- navigation and grouping reflect actual tasks
- comparison, filtering, or scanning needs are handled
- information density fits the usage context
- content does not bury risk, cost, or irreversible actions

Plan obligation for 10/10: hierarchy, grouping, and labels tied to user decisions.

## Pass 4: Interaction States

Rate 0-10: Are all meaningful states designed as product behavior?

Evaluate:

- loading and pending states
- empty and first-use states
- partial, stale, delayed, or uncertain states
- success and confirmation states
- disabled or unavailable actions
- account, permission, quota, or environment constraints

Plan obligation for 10/10: state matrix with user-facing copy, available actions, and verification for each meaningful state.

## Pass 5: Recovery And Re-entry

Rate 0-10: Can users recover from mistakes, failures, interruptions, and return visits?

Evaluate:

- validation and undo where appropriate
- retry, cancel, back, refresh, resume, reopen
- saved progress or clear reset behavior
- failed external dependency or network issue
- error messages with cause and next step
- support or escalation path

Plan obligation for 10/10: recovery paths for every realistic failure or interruption scenario.

## Pass 6: Trust, Safety, And Clarity

Rate 0-10: Does the experience earn user confidence before asking for trust?

Evaluate:

- source, confidence, or uncertainty is visible where needed
- consequences of actions are clear
- irreversible or high-stakes actions require appropriate confirmation
- privacy, cost, data use, or permissions are explained in context
- AI-generated or automated output includes enough rationale and correction path
- tone is calm, specific, and credible

Plan obligation for 10/10: trust cues and consequence copy at the moments where users decide whether to proceed.

## Pass 7: Accessibility And Responsive Use

Rate 0-10: Can users operate the flow across abilities, devices, and input modes?

Evaluate:

- keyboard navigation
- focus order and visible focus
- screen reader labels and semantic structure
- contrast and text sizing
- touch target sizes
- mobile, tablet, and desktop path differences
- reduced motion or time pressure concerns when relevant

Plan obligation for 10/10: accessibility acceptance criteria and responsive behavior for the primary scenarios.

## Pass 8: Measurement And Feedback Loops

Rate 0-10: Can the team tell whether the experience works after shipping?

Evaluate:

- task completion can be measured
- time to first value can be measured
- drop-off and recovery points are observable
- user feedback or support signal exists
- privacy-safe analytics are named when needed
- qualitative review or usability follow-up is planned

Plan obligation for 10/10: measurable UX acceptance criteria and a post-ship audit path.
