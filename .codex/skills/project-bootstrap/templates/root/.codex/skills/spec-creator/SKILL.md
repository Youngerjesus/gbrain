# Spec Creator

Use this skill to turn an accepted feature direction into a decision-complete specification package.

## Output

- `spec.md`
- `contracts.md`

## Rules

- Use `SPEC_TEMPLATE.md` for the spec.
- Use `CONTRACT_TEMPLATE.md` for the completion contract.
- Prefer measurable, verifiable requirements.
- Avoid vague adjectives unless backed by a rubric or proof method.
