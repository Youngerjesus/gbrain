# Live UX Review Sections

Use these eight passes after target discovery. Each pass must state evidence, score, and gaps. For dimensions that cannot be directly exercised, mark the method as `INFERRED` or `BLOCKED` instead of guessing.

## Pass 1: First Value And Onboarding

Rate 0-10: Can the target user reach first meaningful value quickly and confidently?

Evaluate:

- entry point matches intent
- first screen makes the product state understandable
- setup asks only necessary questions
- progress appears early
- next action is obvious
- first value is visible and named

Fix direction: reduce steps, remove unnecessary decisions, and make the first useful outcome unmistakable.

## Pass 2: Core Task Flow

Rate 0-10: Can the user complete the main task without unnecessary detours?

Evaluate:

- start, middle, and finish
- primary action clarity
- feedback after each action
- completion confirmation
- ability to continue or exit naturally
- task flow on mobile and desktop when relevant

Fix direction: remove ambiguity about what happened and what the user should do next.

## Pass 3: Information Architecture

Rate 0-10: Is information organized around user decisions and attention?

Evaluate:

- most important information appears first
- labels match user language
- grouping supports the task
- scanability and comparison needs
- risk, cost, or irreversible consequences are not buried

Fix direction: reorder and relabel around the user's next decision.

## Pass 4: Interaction States

Rate 0-10: Are meaningful states visible, understandable, and actionable?

Evaluate:

- loading and pending
- empty and first-use
- partial, stale, delayed, or uncertain
- success and confirmation
- disabled or unavailable actions
- account, permission, quota, or environment constraints

Fix direction: every state should tell the user what is happening and what they can do.

## Pass 5: Recovery And Re-entry

Rate 0-10: Can users recover from mistakes, failures, interruptions, and return visits?

Evaluate:

- validation, undo, retry, cancel, and back
- refresh, resume, reopen, or saved progress
- failed external dependency or network issue
- error messages with cause and next step
- support or escalation path

Fix direction: make failure survivable and make returning unsurprising.

## Pass 6: Trust, Safety, And Clarity

Rate 0-10: Does the experience earn confidence before asking for trust?

Evaluate:

- source, confidence, or uncertainty
- consequences of actions
- confirmation for high-stakes operations
- privacy, cost, data use, or permission explanation
- AI-generated output rationale and correction path when relevant
- tone, specificity, and credibility

Fix direction: place trust cues at the decision point, not in a buried policy page.

## Pass 7: Accessibility And Responsive Use

Rate 0-10: Can users operate the flow across abilities, devices, and input modes?

Evaluate:

- keyboard navigation
- focus order and visible focus
- semantic labels and screen reader hints
- contrast and text size
- touch targets
- mobile, tablet, and desktop behavior
- reduced motion or time-pressure concerns

Fix direction: verify accessibility in the flow, not only component by component.

## Pass 8: Measurement And Feedback Loops

Rate 0-10: Can the team tell whether the experience works after shipping?

Evaluate:

- task completion signal
- time to first value
- drop-off and recovery points
- support or feedback entry point
- privacy-safe analytics where needed
- usability follow-up or periodic review

Fix direction: instrument or observe the points where users hesitate, fail, or recover.
