# Spec Reviewer

Use this skill to pressure-test a spec package before task breakdown or implementation.

## Review Focus

- ambiguity
- feasibility
- scope drift
- missing edge cases
- weak completion evidence

## Approval Gate

Do not approve while blocker findings remain.

A spec is ready only when:
- task planners can derive work without inventing scope
- implementers can build without asking what counts as compliant
- reviewers can judge pass/fail from the written docs and required evidence
