# Project Task Memory

`projects/` stores thin human-readable task memory for work that spans Codex
sessions. It is not a replacement for requirement contracts, plans, progress, or
evidence files.

Use:

- `projects/task-card-template.md` for the global Task Card shape.
- `projects/<project-id>/task-card.md` for a specific project or initiative.

Each project Task Card should stay short enough to read before a session starts.
Long source material belongs in the linked requirement, sequence, progress,
decision, and evidence files. The card should carry only the current outcome,
inputs, constraints, verification standard, and durable decisions needed to keep
the next agent from restarting from scratch.
